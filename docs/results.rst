Definitions:

- *FINAL score*: The overall terminal emulator quality score, calculated as
  the average of all feature scores (WIDE, LANG, ZWJ, VS16, VS15, and DEC Modes),
  then scaled (normalized 0-100%) relative to all terminals tested. Higher scores
  indicate better overall Unicode and terminal feature support. This score excludes
  TIME performance metrics to focus purely on feature completeness.
- *WIDE score*: Shows the Unicode version specification most closely
  matching in compatibility (highest version with 90% match or greater),
  followed by the scaled percentage score (determined by version release
  level multiplied by the pct of wide codepoints supported at that version).
  Format: version/percentage (e.g., 16.0.0/86.4%).
- *LANG score*: The percentage of international languages tested
  as having support, scaled.
- *ZWJ score*: Determined by version release level of emoji sequences
  with Zero-Width Joiner support, multiplied by the pct of emoji
  sequences supported at that version, scaled.
- *VS16 score*: Determined by the number of Emoji using Variation
  Selector-16 supported as wide characters, scaled.
- *VS15 score*: Determined by the number of Emoji using Variation
  Selector-15 supported as narrow characters, scaled.
- *Mode 2027*: DEC Mode 2027 (GRAPHEME_CLUSTERING) support. Shows 'enabled'
  if the mode is currently enabled, 'may enable' if the mode is supported but
  not enabled and can be changed to enabled, or 'no' if not supported.
  This mode enables grapheme clustering behavior in the terminal.
- *DEC Modes score*: Determined by the number of DEC private modes
  that are changeable by the terminal, scaled. The number shows
  the total count of changeable modes, with a link to detailed results.
- *TIME score*: Determined by test execution time in seconds, scaled
  inversely (lower time is better). The value shows elapsed seconds
  during the test run, with a link to detailed timing information.
