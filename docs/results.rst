Definitions:

- *FINAL score*: The overall terminal emulator quality score, calculated as
  the average of all feature scores (WIDE, LANG, ZWJ, VS16, VS15, and DEC Modes),
  then scaled (normalized 0-100%) relative to all terminals tested. Higher scores
  indicate better overall Unicode and terminal feature support. This score excludes
  TIME performance metrics to focus purely on feature completeness.
- *WIDE score*: Overall percentage of wide character codepoints correctly
  displayed across all Unicode versions tested. Calculated as the total
  number of successful codepoints divided by total codepoints tested, scaled.
- *LANG score*: Calculated using the geometric mean of success percentages
  across all international languages tested. This fairly accounts for partial
  support (e.g., 99%, 98%) without letting one low score dominate, scaled.
- *ZWJ score*: Overall percentage of emoji ZWJ (Zero-Width Joiner) sequences
  correctly displayed across all emoji versions tested. Calculated as the total
  number of successful sequences divided by total sequences tested, scaled.
- *VS16 score*: Determined by the number of Emoji using Variation
  Selector-16 supported as wide characters.
- *VS15 score*: Determined by the number of Emoji using Variation
  Selector-15 supported as narrow characters.
- *Mode 2027*: DEC Mode 2027 (GRAPHEME_CLUSTERING) support. Shows 'enabled'
  if the mode is currently enabled, 'may enable' if the mode is supported but
  not enabled and can be changed to enabled, or 'no' if not supported.
  This mode enables grapheme clustering behavior in the terminal.
- *DEC Modes*: Determined by the number of DEC private modes
  that are changeable by the terminal, scaled.
- *Elapsed Time*: Test execution time in seconds, scaled inversely
  (lower time is better).
